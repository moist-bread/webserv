More about this page:

Unlike the Post a Picture page here the upload is made simply via an Html form, no special script. So let's take a deeper loop at how the form sends it's content.
For starters to send a file or multiple inputs (e.g. text) your form needs to send it as a multipart/form-data. The multipart form-data is sent in a Request's body and is structured as follows:

------WebKitFormBoundaryAw0OjJFbA0LRxyB0
Content-Disposition: form-data; name="media"; filename="example.png"
Content-Type: image/png

**bytes representing the image**
------WebKitFormBoundaryAw0OjJFbA0LRxyB0
Content-Disposition: form-data; name="text"

example text
------WebKitFormBoundaryAw0OjJFbA0LRxyB0--

The main components are: the Boundary String, The Headers for each part of the data and the Data itself.

Boundary String: A string of up to 70 characters that serves as a separator for each part of the data and as footer at the end. To know what the Boundary is you need to go check the Request Headers, not to be confused with the multipart/form-data headers, where the Contente-Type header will indicate it.

Example:
Content-Type: multipart/form-data; boundary=----Exampl3Aw0OjJFbA0LRxyB0

Multipart Headers: These headers give information about the following Data, such as the Content-Type or Content-Disposition

Data: Acording to the headers we decide how to handle this data. For this server whenever the Content-Disposition contains a "filename" parameter we save the file in the upload_store folder defined in the configuration file and then register that we saved it in a "database" file in the same folder, in a json format.
