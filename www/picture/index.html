More about this page:

Here you can see an example of how to upload images using Javascript inside the Html.
Instead of this Html form doing it's default behaviour we have the Javascript do a POST request as a "middleman" for the Client viewing this page.

Why is that necessary? Well, whenever you try to upload a large file, once it starts it will continue sending the content of the file until the very end, which can be quite inconvenient for the Server, which would need to keep collecting the content of the uploaded file even when it has already surpassed the Upload Limit configured for the Server

To force the Client to stop uploading the common approach is to disconnect the client, but doing so would leave the User looking at the website quite confused having suddenly lost connection without a warning. That's where the Javascript "middleman" comes in, if the execution of the Javascript is the one that gets it's connection closed it can still give some informative explanation to the User as to what happened, making for a clearer upload experience
