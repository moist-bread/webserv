For this Server we also implemented Range request, allowing for video streaming and more.

The way it works is that when you try to play a video it will send a GET request that has a "Range" header. The header will then specify what part of the content it wants to GET. There are 4 ways that it can be formatted but let's go with the simplest example:

Range: bytes=0-

What that means to the Server is that the Client wants all the bytes of the requested file from the byte 0 up until the end. The Server can then respond with an amount of bytes but has to specify which bytes were they and how big is the whole content, for example:

Content-Range: bytes 0-100/5000

Which means the request is sending all bytes from 0 until 100 and that the total is 5000. Afterwards the Client can ask for the remaining bytes when it needs them.
